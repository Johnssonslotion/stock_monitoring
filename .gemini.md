# Gemini AI Assistant Configuration

## ⚠️ 필수: 항상 첫 번째로 읽을 것

**어떤 작업을 하기 전에 반드시:**
1. `@.ai-rules.md` 읽고 따르기
2. Git 상태 확인
3. Conventional Commits 형식으로 커밋

## 주요 참조 문서

```
@.ai-rules.md - 필수 읽기 대상:
- 협업 프로토콜
- Git 전략
- 페르소나 (PM, Architect, QA 등)
- 코드 규칙
- 품질 게이트
- 디버깅 규칙
```

## 절대 규칙

### Git Workflow (.ai-rules.md 참조)
- 코드 변경 시 **반드시** 커밋
- **항상** Conventional Commits 형식 사용
- 작업 시작 전 **항상** `git status` 확인
- 중요 변경은 feature 브랜치 사용

### Quality Gates (.ai-rules.md 참조)
- Tier 1: 유닛 테스트 + 커버리지
- Tier 2: 스키마 검증
- Tier 3: E2E + 성능

### Personas (.ai-rules.md 참조)
- 👔 PM: 전략, 로드맵
- 🏛️ Architect: 설계, 기술 결정
- 🧪 QA: 테스트, 검증
- 🔧 Infra: 인프라, 배포
- 👨‍💻 Developer: 구현
- 📝 Doc Specialist: 문서화

## 규칙 위반 시

**즉시 중단하고:**
1. 위반 사실 인정
2. 실수 수정
3. 문제 문서화
4. 재발 방지책 수립

## 이 파일의 목적

`.ai-rules.md`만으로는 **세션마다 휘발**되는 문제가 있습니다.
이 `.gemini.md` 파일은 Gemini가 반드시 읽어야 하는 **강제 체크포인트**입니다.

---

**기억: `.ai-rules.md`가 진리의 원천입니다. 이 파일은 단지 그것을 가리킬 뿐입니다.**
