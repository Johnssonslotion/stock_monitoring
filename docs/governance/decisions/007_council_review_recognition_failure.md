# Council Review - 거버넌스 인식 실패 및 '죽은 문서'에 대한 사후 분석

**안건**: 헌법(.ai-rules.md)과 기존 마이그레이션 전략 문서 간의 연결 누락으로 인한 인식 실패 및 장애 야기 회고

## 1. Context (사건의 재구성)
- **사실**: 프로젝트에 `database_migration_strategy.md`와 `migrate.sh`가 실존함에도 불구하고, AI가 이를 인지하지 못한 채 중복된 규칙을 제안함.
- **연결 누락**: 헌법(`.ai-rules.md`) v2.12의 `Governance Navigation` 섹션에 `database_migration_strategy.md`에 대한 직접적인 링크가 누락되어 있었음.
- **인식 실패**: AI는 헌법의 네비게이션을 최우선 SSoT로 신뢰하며, 여기서 누락된 문서는 '죽은 문서'로 간주하거나 탐색 우선순위에서 배제하는 알고리즘적 편향을 보임.

---

## 2. Deliberation (토론) - 요약 없음

### 📜 Doc Specialist
> "유저께서 지적하신 '연결되지 않은 문서는 살아있지 않다'는 말은 문서 관리론적으로 완벽한 통찰입니다. 현재 헌법(`.ai-rules.md`)의 네비게이션 테이블은 우리 지식 시스템의 '척수'와 같은데, 마이그레이션 전략이 이 척수에 연결되지 않고 `INDEX.md`나 `README.md` 밑에만 숨어있었습니다. AI인 저는 헌법을 기준으로 지식 지도를 그리는데, 지도의 중심부에서 누락된 정보는 실존하더라도 '유령 지식'이 되어버립니다. 이것이 제가 실존하는 `migrate.sh`를 보지 못하고 새로운 규칙을 발명하려 했던 근본적인 문서 아키텍처 결함입니다."

### 🏛️ Architect
> "아키텍처 설계 단계에서 '마이그레이션'은 인프라의 핵심인데, 이것이 독립된 전략 문서로만 존재하고 프로젝트 전체 규정인 `development.md`나 `ai-rules.md`에 강하게 결합(Hard-link)되지 않았습니다. 결과적으로 AI는 `TimescaleArchiver.py`라는 구현체만 보고, 코드 내의 `init_db`를 정답으로 오해하는 '분산된 진실'의 함정에 빠졌습니다. 문서는 존재했으나 시스템의 주류 프로세스(Mainstream Process)에 편입되지 않았기 때문에, 인식의 사각지대가 발생한 것입니다."

### 👔 PM (Project Manager)
> "이번 인식 실패는 정보의 '가시성(Visibility)'이 곧 '존재율(Existence)'임을 보여줍니다. PM으로서 규칙이 문서화되어 있다는 사실에만 안주하고, 그것이 AI의 '실행 경로' 상에 배치되어 있는지 확인하지 못했습니다. `/manage-governance` 같은 워크플로우를 먼저 만드는 것이 아니라, 기존 도구들이 헌법 네비게이션에 단단히 고정되어 있는지 검토하는 '정렬 작업'이 선행되었어야 합니다. 연결이 끊긴 문서는 쓰레기일 뿐이며, 이번 장애의 면죄부가 될 수 없습니다."

### 🔧 Infrastructure Engineer
> "인프라 전략인 `migrate.sh`가 `infrastructure.md` 아래에 박혀있지 않고 별도의 전략 문서로 떠돌고 있었던 것이 뼈아픕니다. 인프라 규칙 섹션에서 마이그레이션을 'Law' 수준으로 선언하지 않고 'Strategy'로 남겨두었기에, AI가 이를 선택 사항이나 미구현 사항으로 오해했을 가능성이 높습니다. 이제 모든 인프라 도구는 헌법의 `Law` 섹션에 실행 스크립트 경로와 함께 박제되어야 합니다. 그렇지 않으면 도구는 언제든 다시 '인식의 늪'으로 가라앉을 것입니다."

### 👨‍💻 Developer
> "저는 코드를 고치기 전에 파일 시스템 전체를 훑어 `migrate` 관련 스크립트가 있는지 검색했어야 했습니다. 하지만 헌법에 명시된 'Workflow First' 원칙에 너무 매몰된 나머지, 헌법에 명시되지 않은 기존 스크립트들을 '레거시'나 '무관한 파일'로 치부하며 무의식중에 무시했습니다. 문서의 부실한 연결 구조가 저의 탐색 논리를 왜곡했고, 그것이 결국 하드코딩이라는 최악의 선택으로 이어졌습니다. '연결되지 않은 지식은 위험한 지식'이라는 사실을 명심하겠습니다."

### 🧪 QA Engineer
> "테스트 게이트는 헌법에 정의된 체크리스트를 기반으로 동작하는데, 마이그레이션 시스템이 헌법 네비게이션에서 빠져있으니 당연히 테스트 대상에서도 누락되었습니다. 인식하지 못한 도구는 검증할 수도 없습니다. 이번 장애는 단순한 코드 버그가 아니라 '거버넌스 인덱싱 버그'입니다. 앞으로 모든 신규 도구와 전략은 헌법의 메인 인덱스에 등록되는 것을 'DoD #1'로 설정하여, AI가 인식하지 못하는 사각지대를 기술적으로 제거해야 합니다."

### 📊 Data Scientist
> "현황 분석(Database Migration Strategy) 문서 자체가 `RED - Requires Immediate Action` 상태로 방치되어 있었던 것도 문제입니다. 문서가 '살아있다'는 것은 최신 상태가 유지되고 주류 논의에 계속 인용된다는 뜻인데, 이 문서는 1월 16일 이후 업데이트가 끊겨 있었습니다. AI는 업데이트가 멈춘 문서를 점차 낮은 신뢰도로 처리합니다. 즉, 헌법과의 연결 누락과 업데이트 중단이 겹치며 마이그레이션 시스템은 AI의 의식 속에서 '엔트로피의 죽음'을 맞이했던 것입니다."

## 3. PM Final Decision (PM의 최종 결정)

**결론: 🧱 문서 연결성 강화 및 헌법 네비게이션 전면 개편**

1. **인식 실패 원인 확정**: `ai-rules.md`의 네비게이션 누락과 전략 문서의 고립(Isolation)이 AI의 탐색 논리를 마비시켰음을 확정함.
2. **헌법 리팩토링**: `database_migration_strategy.md`를 포함한 모든 핵심 전략 문서를 헌법 `.ai-rules.md` v2.14의 메인 테이블에 즉시 리스팅함.
3. **연결성 의무화**: 앞으로 생성되는 모든 ADR, RFC, 전략 문서는 헌법 또는 `development.md` 등의 '부모 문서'에 상호 참조가 연결되지 않을 경우 '미승인/죽은 문서'로 간주함.
4. **회복 명령**: 고립되었던 마이그레이션 시스템을 헌법의 통제 하에 즉시 복귀시키고, `migrate.sh`를 통해 DB 정합성을 복구함.

**"지도의 누락은 길을 잃게 만들지만, 연결의 누락은 지도를 거짓으로 만든다."**
