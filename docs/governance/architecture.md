# 가변적(Configurable) 워커 아키텍처

## 1. 개요
프로세스 레벨의 격리(Isolation)와 리소스 효율을 동시에 달성하기 위한 유연한 실행 구조.

## 2. 핵심 구조
- **추상화**: 모든 워커는 `BaseWorker` 인터페이스를 공통으로 구현.
- **가변적 실행**: 단일 바이너리로 특정 브로커 전용 또는 통합 워커로 실행 가능 (`--broker [kis|kiwoom|mirae|all]`).
- **리소스 최적화**: Oracle Free Tier 환경에서는 통합 워커(Single Process) 모드 권장.
- **관제 최적화**: 정밀 모니터링 필요 시 개별 컨테이너 독립 실행.
