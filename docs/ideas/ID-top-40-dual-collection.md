# IDEA: Top 40 Dual Collection Candidates (KIS + Kiwoom)
**Status**: 💡 Seed (Idea)
**Priority**: P2

## 1. 개요 (Abstract)
사용자 요청에 따라 **40개 핵심 종목**을 선정하여 KIS와 Kiwoom으로 역할을 분담합니다.
- **KIS**: 40개 종목의 **실시간 틱(Tick)** 데이터 수집 (속도/안정성)
- **Kiwoom**: 40개 종목의 **호가(Orderbook)** + **핵심 섹터 ETF** 집중 수집 (시장 심도/트렌드)

## 2. 선정 기준 및 전략 (Maximized Strategy: 40+30)

### A. KIS: Core Stability (40)
- **대상**: Top 40 Stocks
- **역할**: **Tick 전담** (초고속/안정성)

### B. Kiwoom: Full Capacity (Total 70 Symbols / 100 Slots)
사용자 피드백("40개 호가 + 30개 신규 틱/호가")을 반영하여, Kiwoom의 WebSocket 슬롯 한계(약 100개)를 **최대치로 활용하는 전략**입니다.

- **구성 (Total 100 Slots)**:
    1.  **Core 40**: Top 40 종목 -> **호가(Orderbook) Only** (40 Slots)
        - *틱은 KIS가 담당하므로 중복 제거*
    2.  **Sector Rotation 30 (New)**: Core 40에 포함되지 않은 순환매 대장주 -> **틱 + 호가** (60 Slots)
        - *완전한 데이터셋으로 섹터/테마의 초기 수급 포착*
- **Coverage**: 총 **70개 종목** (Core 40 + New 30)

## 3. Council Review: Feasibility Check (100 Slots)

> **🏗️ Infra**: "Kiwoom OpenAPI의 실시간 등록 한계는 화면번호당 100개입니다. 40(호가) + 30(틱) + 30(호가) = **정확히 100개**입니다. 이론상 가능하지만 여유 슬롯이 0개이므로, 서버 부하 관리가 중요합니다. (Redis Sharding 필수)"

> **🕵️ Data Scientist**: "훌륭합니다. **Core 40**은 안정적으로 방어(SIS)하고, **New 30**은 공격적으로 순환매(Sector Rotation)를 탐지할 수 있습니다. 특히 New 30에 틱 데이터가 포함되어야 '세력의 진입(체결강도 급증)'을 포착할 수 있으므로 이 전략이 분석 관점에서 최적입니다."

> **👔 PM**: "Risk-Reward가 확실하네요. **Limit(100)**를 꽉 채우는 공격적인 운용을 승인합니다. 단, 장애 발생 시 'New 30' 틱 수집을 우선 드롭하는 **Degradation Plan**을 마련합시다."
