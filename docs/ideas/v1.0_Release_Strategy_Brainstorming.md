# Brainstorming: v1.0 "Professional Trading Terminal" Release Strategy

## 🎯 Goal: What is v1.0?
v1.0은 단순한 데이터 수집기를 넘어, **"신뢰할 수 있는 데이터 위에서 전략을 시뮬레이션하고 시장을 감시할 수 있는 완결된 도구"**여야 합니다.

## 🔍 Gap Analysis (현재 vs v1.0)

| 항목 | 현재 상태 | v1.0 요구사항 | 결핍 (Gap) |
| :--- | :--- | :--- | :--- |
| **Data** | 99% 무결성 (98종목) | 100% 무결성 + 자동 복구 | 과거 데이터 전수 자동 복구 (Historical Backfill) |
| **UI/UX** | 기본 차트 + 맵 | 프리미엄 테마 + 고급 분석 도구 | Chart V2 (VWAP, 구간 통계), 도구 모음 디자인 |
| **Trading** | 관측 위주 (Observation) | 가상 체결 및 잔고 관리 | Virtual Exchange (Pillar 6) 실구현 |
| **Strategy** | 인프라만 존재 | 실행 가능한 실전 전략 통합 | 샘플 전략 1-2종의 실시간 시각화 반영 |
| **Stability** | 수동 검증 위주 | 자가 치유 (Self-Healing) | Chaos Testing 시스템화 (Pillar 4 Phase 3) |

---

## 🏛️ Council of Six - v1.0 브레인스토밍 의견

### 👔 PM (Project Manager)
> "v1.0의 핵심은 **'사용자 경험의 완결성'**입니다. 데이터가 아무리 정확해도 쓸모 있는 전략이나 체결 시뮬레이션이 없다면 '터미널'이라 부를 수 없습니다. 우선순위를 **[Pillar 6: Virtual Exchange]**의 MVP 구현과 **[Pillar 3: Chart V2]**의 시각적 완성도에 두어야 합니다. 이번 Q1-B 마일스톤 내에 '가상 매수/매도' 버튼 하나라도 동작하게 만드는 것이 비즈니스적으로 큰 가치가 있습니다."

### 🏛️ Architect
> "기술적으로 v1.0은 **'중앙화된 명령 체계'**를 갖춰야 합니다. 현재 `verification-worker`가 성공적으로 통합되었듯이, 전략 실행과 가상 체결 로직도 `StrategyEngine`으로 통합되어야 합니다. 특히 `VirtualExchange`는 실제 브로커 API와 100% 호환되는 인터페이스를 가져야 나중에 실전 매매로 전환할 때 코드 수정을 최소화할 수 있습니다. 이것이 v1.0의 아키텍처적 완성도입니다."

### 🔬 Data Scientist
> "데이터 측면에서 v1.0은 **'Ground Truth에 대한 맹신'**을 가능케 해야 합니다. 현재 소급 검증은 'SKIPPED' 처리된 부분이 많은데, v1.0에서는 과거 분봉 API를 연동하여 '과거 어느 시점이든 100% 정확한 데이터'를 보장해야 합니다. 또한, 틱 데이터의 미시구조(Micro-structure) 분석 기능이 분석 툴로서의 차별점을 만들어줄 것입니다."

### 🔧 Infrastructure Engineer
> "운영 관점에서 v1.0은 **'Zero OS 관리'**를 지향해야 합니다. 지금처럼 수동으로 스크립트를 돌리는 것이 아니라, 모든 것이 Docker-compose와 Crontab 내에서 자가 치유되어야 합니다. 또한 태스크가 98종목으로 늘어난 만큼, 리소스 사용량(CPU/RAM)이 쿼드코어 24GB 제한 내에서 안정적으로 유지되는지 스트레스 테스트가 선행되어야 합니다."

### 👨‍💻 Developer
> "구현 관점에서는 **'사용성(Usability)'**입니다. 차트에서 마우스를 움직일 때의 지연시간, 심볼 전환 속도 등이 '프로페셔널'한 느낌을 주어야 합니다. 현재 프론트엔드의 Chart V2를 마무리하여 다양한 기술적 지표(VWAP, 이동평균선 등)를 레이어 형태로 쉽게 켜고 끌 수 있게 만드는 것이 급선무입니다."

### 🧪 QA Engineer
> "v1.0의 라벨을 붙이려면 **'공포 없는 배포'**가 가능해야 합니다. 오늘 NameError로 인한 장애가 있었던 것처럼, 배포 자동 검증(RFC-006)을 넘어, `Chaos Mesh` 같은 도구를 활용해 강제로 네트워크를 끊거나 DB를 과부하시켜도 시스템이 데이터 유실 없이 복구되는지 증명해야 합니다. 이 증명서(Test Registry 100% Pass)가 v1.0의 보증수표가 될 것입니다."

## 💡 Proposed v1.0 Checklist (추가 과제)

1. **[Pillar 6] Virtual Exchange MVP**: 가상 매수/매도, 잔고 동기화, 슬리피지 모델 적용.
2. **[Pillar 3] Premium Dashboard**: Chart V2 완성 (Indicator Layers), Dark Mode 고도화.
3. **[Pillar 2] Deep Integrity**: 과거 1년치 전 종목 자동 소급 복구 시스템 완성.
4. **[Pillar 4] Full Observability**: Grafana/Prometheus (또는 경량 대시보드) 기반 실시간 경보 완성.
5. **[Strategy] Starter Kit**: '돌파 매매' 등 기본 전략 1종 탑재 및 가상 자산 변화 추적.
